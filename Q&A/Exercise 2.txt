Exercise 2.1 Какво ще се изпринти на конзолата и защо?
{			
 {
  {
var a = 1;
  }
 }
}
console.log(a);
-конзолата изпринтва 1, защото дадената променлива е декларирана от тип var, а тя зачита само обхвата на функция и 
 глобалния обхват. Затова стойността остава същата като декларираната такава, защото не зачита обхвата на блок от код,
 какъвто е даден в този пример. 

Exercise 2.2 Какво ще се изпринти на конзолата и защо?
{
 {
  {
let a = 1;
  }
 }
}
console.log(a);  
-конзолата връща грешка от тип "Uncaught ReferenceError: a is not defined", което е JavaScript изключение и 
 означава „променливата не е дефинирана“, когато някъде има препратка към несъществуваща променлива, какъвто е случаят
 тук, защото променливата е декларирана от тип let, а тя зачита всички обхвати на блок от код, дори и на if блок от код,
 затова при изпринтване на дадената променлива връща грешка, защото променливата се намира в обхвата на блок от код и извън
 този обхват не съществува или с други думи не е декларирана.
 
Exercise 2.3 Какво ще се изпринти на конзолата и защо?
function foo(){
 {
  {
let a = 1;
var b = "b";
  }
 }
}
foo();
console.log(a);
console.log(b);
-конзолата връща две грешки от тип "Uncaught ReferenceError: a is not defined" и "Uncaught ReferenceError: b is not
 defined", които са JavaScript изключения и означават „променливата не е дефинирана“, когато някъде има препратка към 
 несъществуваща променлива. Променливата "а" е декларирана от тип let, а тя зачита всички обхвати на блок от код, дори на
 if блок от код както и обхват на функция, затова при изпринтване на дадената променлива връща грешка, защото променливата
 се намира в обхвата на блок от код и извън този обхват не съществува или с други думи не е декларирана. Променливта "b" 
 е декларирана от тип var, а тя зачита само обхвата на функция и глобалния обхват, в случая тя е декларирана в обхвата на
 функция, затова връща грешка, защото извън нея тя не съществува или не е декларирана. 

Exercise 2.4 Какво ще се изпринти на конзолата и защо?
var a = "hi";
function bar (){
 {
  {
console.log(a);
  }
 }
}
-ако се добави if (false) {var a = 200;}, което не би трябвало да се изпълни, но в този случай променливата "а" е
 декларирана глобално и е от тип var, а тя зачита само обхвата на функция и глобалния обхват. Тук става въпрос за Hoisting
 в JavaScript, което е поведение, при което функция или променлива може да се използва преди декларация.В случая вътре в
 обхвата на функция никъде не е декларирана променливата "а", което означава, че JavaScript я декларира автоматично от тип
 var и я изкачва (Hoisting) горе в началото на функцията със стойност undefined, защото само този тип данни може да има
 "hoisting", но тъй като няма присвоена стойност, конзолата изпринтва undefined. Но ако се махне този if, конзолата 
 изпринтва hi. Това би довело до множество грешки, тъй като if не се изпълнява заради условието, следователно не би
 трябвало да повлияе на кода по никакъв начин, но повлиява с два различни резултата.
